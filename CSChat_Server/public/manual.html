CSChat 관리자 매뉴얼

CSChat 관리자 페이지의 모든 기능을 안내합니다. 사용자 관리부터 백업까지 단계별로 설명합니다.

📅 최종 업데이트: 2026년 3월 적용 서버 버전: CSChat Server v2.0.0 관리자 페이지: /admin/index.html

개요

CSChat 관리자 페이지는 회사 내부 채팅 서버를 효율적으로 운영하기 위한 통합 관리 도구입니다. 별도의 앱 설치 없이 웹 브라우저에서 접근 가능하며, 다음과 같은 핵심 기능을 제공합니다.

실시간 대시보드

접속자 현황, 메시지 통계, 트래픽 차트를 한눈에 확인합니다.

사용자 관리

계정 생성·삭제·수정, 관리자 권한 부여, MAC 기기 바인딩을 설정합니다.

대화방 모니터링

전체·개인 대화방의 메시지(텍스트·이미지·영상·파일)를 실시간으로 열람합니다.

긴급 공지

전체 사용자에게 긴급 공지를 발송하고, 확인 여부를 추적합니다.

시스템 로그

로그인 시도, 백업, 공지 등 모든 시스템 이벤트를 조회하고 삭제합니다.

백업 & 복구

즉시·자동 데이터베이스 백업, 백업 파일 조회 및 삭제를 지원합니다.

로그인

브라우저에서 http://[서버 IP]:3001/admin/index.html 에 접속합니다.
관리자 아이디와 비밀번호를 입력하고 로그인 버튼을 클릭합니다.
로그인 성공 시 대시보드 화면으로 자동 전환됩니다.
관리자 계정이 없는 경우, 서버 관리자에게 계정 생성을 요청하거나 데이터베이스에서 직접 관리자 권한을 부여해야 합니다.

대시보드

READ-ONLY

로그인 직후 표시되는 화면으로, 서버의 전체적인 현황을 한눈에 파악할 수 있습니다.

전체 사용자 수

등록된 계정의 총 개수를 표시합니다.

현재 접속자

현재 실시간으로 접속 중인 사용자 수입니다.

총 메시지

전체 대화방에 쌓인 누적 메시지 수입니다.

대화방 수

생성된 전체 대화방(전체·1:1 포함)의 수입니다.

일별 트래픽 차트

최근 7일간의 메시지 발송 현황이 막대 차트로 표시됩니다. 특정 날짜에 트래픽이 급증했는지 한눈에 파악할 수 있습니다.

최근 시스템 로그

가장 최근에 발생한 시스템 이벤트 5건이 표시됩니다. 상세 조회는 시스템 로그 메뉴를 이용하세요.

상단 우측의 아이콘을 클릭하면 라이트/다크 테마를 전환할 수 있습니다.

사용자 관리

사용자 등록

우측 상단 「+ 신규 사용자」 버튼 클릭
아이디와 초기 비밀번호 입력 후 「등록하기」 클릭

엑셀 일괄 등록

여러 사용자를 한 번에 등록할 때 사용합니다.

  • 엑셀 파일 형식: 1열 = 사용자명, 2열 = 비밀번호
  • 「엑셀 업로드」 버튼으로 파일 선택 시 자동 처리
  • 중복 아이디는 건너뛰고 성공/실패 건수를 안내합니다.

사용자 정보 수정

사용자 목록에서 설정 아이콘 클릭 시 수정 모달이 열립니다. 수정 가능한 항목은 다음과 같습니다.

항목 설명
사용자명 로그인에 사용하는 아이디를 변경합니다.
관리자 권한 토글 스위치로 관리자 권한을 부여/회수합니다.
기기 고유 식별자 (MAC) 특정 기기에서만 로그인을 허용할 때 사용합니다.
기기 식별자 매핑 강제 활성화 시 등록된 기기 외 접속을 차단합니다.
비밀번호 초기화 비밀번호를 1234로 초기화합니다.
비밀번호 변경 새 비밀번호 + 확인 입력 후 「비밀번호 변경 적용」 클릭. 4자 이상 필수, 불일치 시 오류 표시. 변경 성공 시 입력 필드 자동 초기화.

사용자 삭제

목록의 아이콘 클릭 → 확인 모달에서 「확인」을 눌러 영구 삭제합니다.

사용자를 삭제하면 해당 사용자의 계정이 즉시 비활성화되며 복구할 수 없습니다.

정렬 기능

테이블 헤더(사용자명, 가입일, 접속일 등)를 클릭하면 오름차순/내림차순으로 목록을 정렬할 수 있습니다.

대화방 모니터링

모든 대화방의 메시지를 실시간으로 모니터링하고, 관리 작업을 수행합니다.

대화 내용 조회

좌측 대화방 목록에서 확인할 방을 클릭합니다.
우측 패널에 해당 방의 메시지 목록이 시간순으로 표시됩니다.
이미지·영상·파일은 인라인 미리보기로 바로 확인할 수 있습니다.
이미지 미리보기

채팅 내 첨부 이미지를 클릭하면 원본 크기로 새 탭에서 열립니다.

동영상 재생

영상 파일을 모니터링 화면에서 직접 재생할 수 있습니다.

파일 다운로드

첨부 파일 링크를 클릭하여 파일을 직접 다운로드할 수 있습니다.

참여자 조회

방 카드의 「N명 참여」를 클릭하면 참여자 목록과 온라인 상태를 확인합니다.

대화 내용 초기화

대화방 카드의 초기화 클릭 → 확인 후 실행.
대화방과 참여자 정보는 유지되며, 메시지만 영구 삭제됩니다.

대화방 삭제

대화방 카드의 삭제 클릭 → 확인 후 실행.
메시지·참여자 정보 모두 영구 삭제됩니다. 「전체대화방」은 삭제할 수 없습니다.

메시지는 소켓(Socket.IO)으로 실시간 수신됩니다. 새 메시지가 발생하면 화면이 자동으로 갱신됩니다.

긴급 공지 확인 현황

긴급 공지 발송

우측 상단 「긴급 공지」 버튼 또는 상단 헤더의 버튼 클릭
공지 내용 입력 (줄바꿈 가능)
「공지 발송」 클릭 → 전체 대화방에 즉시 브로드캐스트

확인 현황

각 공지 카드에서 확인 인원 / 전체 인원 비율이 실시간으로 표시됩니다.

  • ● 읽음 공지를 확인한 인원 수
  • ● 안 읽음 아직 확인하지 않은 인원 목록
  • 바 그래프로 확인율 시각화

재발송 기능

「상세」 버튼 → 재발송 제어 섹션에서 확인하지 않은 인원에게 재발송할 수 있습니다.

  • 즉시 재발송: 미확인 인원에게 바로 재전송
  • 자동 스케줄: N분 간격으로 자동 재발송 반복 설정 가능
대화방 모니터링 화면의 공지 메시지에서도 「실시간 확인 현황」 버튼을 통해 즉시 현황을 조회할 수 있습니다.

시스템 로그

날짜 필터링

퀵 필터 버튼 또는 날짜 직접 입력으로 기간을 설정한 후 「조회하기」를 클릭합니다.

버튼 조회 범위
오늘 오늘 날짜의 로그만 조회
1주일 오늘 포함 7일 이내 로그
1개월 오늘 포함 1개월 이내 로그
1년 오늘 포함 1년 이내 로그

로그 선택 삭제

삭제할 로그 행의 체크박스를 클릭합니다. (헤더 체크박스로 전체 선택 가능)
툴바에 표시된 「N건 선택됨」과 선택 삭제 버튼 클릭
확인 모달에서 「확인」을 눌러 삭제합니다.

전체 삭제

툴바의 전체 삭제 버튼 클릭 → 현재 DB에 저장된 모든 시스템 로그를 영구 삭제합니다.

로그 레벨

레벨 의미 예시
INFO 정상적인 이벤트 로그인 성공, 백업 생성
WARN 주의가 필요한 이벤트 로그인 실패, 비밀번호 불일치
ERROR 오류 발생 서버 오류, DB 접근 실패
삭제된 로그는 복구할 수 없습니다. 감사(Audit) 목적으로는 삭제 전 백업을 권장합니다.

검색 및 백업

즉시 백업

저장 경로 입력 (비울 경우 서버 내 ./backups 폴더 사용)
「 즉시 백업 실행」 클릭
성공 시 chat[날짜시간].db 형식으로 파일 생성

자동 백업 스케줄

일정 간격으로 자동 백업이 실행되도록 스케줄을 등록합니다.

주기 실행 시점
주 단위 7일 간격
월 단위 30일 간격
분기 단위 분기 말일
년 단위 매년 12월 31일

백업 파일 탐색

백업 파일 카드의 데이터 조회 및 탐색 버튼을 클릭하면 해당 시점의 데이터를 읽기 전용으로 열람할 수 있습니다.

  • 대화방 기록 탭: 백업 시점의 채팅 메시지 미리보기 및 엑셀 출력
  • 시스템 로그 탭: 백업 DB의 시스템 로그 조회 및 엑셀 출력

백업 파일 삭제

백업 카드의 버튼 클릭 → 확인 모달에서 파일명을 확인 후 삭제합니다. 서버 파일시스템에서 영구 삭제됩니다.

「찾아보기」 버튼을 통해 서버 디렉토리를 탐색하여 백업 저장 경로를 손쉽게 선택할 수 있습니다. 경로가 없으면 자동으로 생성됩니다.

자주 묻는 질문

Q. 비밀번호를 잊었습니다.

다른 관리자 계정으로 로그인 후 「사용자 관리」에서 해당 계정의 비밀번호를 1234로 초기화할 수 있습니다. 이후 사용자가 직접 비밀번호를 변경하도록 안내하세요.

Q. 로그가 날짜 필터에도 전부 나옵니다.

날짜 입력 후 반드시 「조회하기」 버튼을 클릭해야 서버에 필터 조건이 전송됩니다. 퀵 필터 버튼(오늘, 1주일 등)을 누르면 자동으로 조회됩니다.

Q. 버튼을 클릭해도 아무 반응이 없습니다.

브라우저 캐시를 비우고 Ctrl + F5로 강력 새로고침 해보세요. 그래도 문제가 지속되면 개발자 도구(F12) 콘솔 탭에서 오류 메시지를 확인하세요.

Q. 서버가 재시작되었을 때 재접속이 안 됩니다.

브라우저를 새로고침하면 자동으로 재연결됩니다. 앱 사용자는 앱 자체가 자동 재연결을 시도합니다.

Q. 특정 사용자의 동일 기기 접속만 허용하려면?

「사용자 관리 → 설정」에서 해당 사용자의 기기 고유 식별자(MAC)를 등록한 뒤 기기 식별자 매핑 강제 옵션을 활성화하면, 등록된 기기 외의 접속이 차단됩니다.

보안 안내

관리자 페이지는 내부 네트워크 전용으로 운영하는 것을 권장합니다. 외부 인터넷에 노출 시 보안 위협이 발생할 수 있습니다.
비밀번호 암호화

모든 비밀번호는 bcryptjs로 해시 처리되어 DB에 저장됩니다. 평문 저장 없음.

MAC 바인딩

사용자별 기기 고유 식별자 등록으로, 인가되지 않은 기기의 접속을 차단합니다.

경로 이탈 방지

백업 파일 삭제 API는 지정 디렉토리 외 경로 접근(Path Traversal)을 서버에서 차단합니다.

정기 백업

자동 백업 스케줄 설정으로 데이터 손실에 대비하세요.

관리자 운영 체크리스트

  • ☐ 관리자 비밀번호는 주기적으로(권장: 90일) 변경
  • ☐ 퇴사자 계정은 즉시 삭제 또는 비활성화
  • ☐ 주간 백업 스케줄 등록 및 백업 파일 확인
  • ☐ 월 1회 시스템 로그 검토 및 이상 징후 확인
  • ☐ 서버 서비스(NSSM) 구동 상태 주기적 점검
